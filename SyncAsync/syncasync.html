Peticiones Síncronas y Asíncronas - AJAX

En la interacción Cliente/Servidor en aplicaciones Web, el cliente pueden realizar dos tipos de peticiones al servidor: Síncronas y Asíncronas.

Una petición Síncrona bloquea la ejecución del código, lo que provoca "congelación" en la pantalla y una experiencia de usuario que no responde.

En una petición Asíncrona el flujo de una página que realiza la petición sigue ejecutandóse mientras el servidor procesa la solicitud y genera la respuesta correspondiente. Esto permite mejorar la experiencia de usuario. A continuación se muestra un ejemploo de una petición asincrona, la cual se especifica con el valor true en el tercer argumento de la función open

const xhr = new XMLHttpRequest(); xhr.open("GET", "./objeto.txt", true); xhr.onload = (e) => { if (xhr.readyState === 4) { if (xhr.status === 200) { console.log(xhr.responseText); } else { console.error(xhr.statusText); } } }; xhr.onerror = (e) => { console.error(xhr.statusText); }; xhr.send(null);
